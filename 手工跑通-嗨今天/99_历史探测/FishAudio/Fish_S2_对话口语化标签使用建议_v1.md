# Fish S2 对话口语化标签使用建议 v1

> 日期：2026-07-01  
> 适用范围：Fish Audio S2 / S2.1 系列，尤其是 `s2.1-pro-free`、`s2.1-pro`、`s2-pro` 的中文父子对话播客。  
> 结论更新：之前 `../../02_音频导演稿/情绪playbook_v1.md` 曾把 Fish 标签策略收窄到少数强声学标签；后续按官方 S2 写法重测后，确认 **自然语言方括号指令可明显影响听感**。本档作为后续 Fish S2 成稿参考。

## 一句话原则

Fish S2 的主语法是 **方括号自然语言指令 + 原始口语台词 + speaker 标签**：

```text
<|speaker:0|>[温和，像自己也没完全想明白] 哎，这个新闻我昨天看了三遍，还是卡住了。
<|speaker:1|>[一下来精神，带着笑意] 啊？还有你看不明白的？
```

不要把标签当“情绪枚举按钮”。更好的写法是把它当给配音演员的简短导演提示：声音大小、语速、情绪方向、说话动机、是否像在想、是否像讲秘密。

## 最新实测结论（2026-07-01）

基于 `音频/标签格式测试/` 的 A/B 结果，后续成稿按以下规则执行：

1. **标签后必须留一个 ASCII 空格。**  
   句首和句中都是 `[标签] 正文` 明显优于 `[标签]正文`。句中写法也要留空格：`我打开盒子，[surprised] 里面真的有一颗小星星！`
2. **主情绪优先用中文自然语言描述。**  
   `[一下子兴奋，声音上扬] 啊？...` 比单个英文抽象标签更容易得到可控、贴中文语境的表现。
3. **不要堆两个以上标签。**  
   `[surprised][excited] 正文` 实测有走样风险。复杂情绪不要堆标签，改成一个中文描述：`[惊讶，又有点兴奋，声音上扬] 正文`。
4. **英文简单标签适合局部声学效果或句中变化。**  
   例如 `[whispering]`、`[sighing]`、`[gasping]`、`[laughing]` 放在句中某个变化点；不要用一串英文标签承担整句主情绪。
5. **全角括号不用。**  
   只用半角 `[]`；`【excited】` 不进入正式脚本。
6. **`s2.1-pro-free` 不再作为正式多说话人一次生成路径。**  
   官方 HTTP TTS API 文档把 `reference_id` 数组 + `<|speaker:n|>` 的多说话人 dialogue 标为 `S2-Pro only`；实测 `s2.1-pro-free` 在换新爸爸 voice id 后出现整段都变成第一个 voice id 的问题。后续没有 `s2-pro` / `s2.1-pro` 稳定额度时，双人正式音频走 `FISH_DIALOGUE_MODE=line_concat`，逐句单 voice id 生成再拼接。

因此，推荐结构是：

```text
<|speaker:1|>[一下子兴奋，声音上扬] 啊？盒子里真的有星星？
<|speaker:0|>我打开盒子，[whispering] 你先别告诉别人。
```

不推荐：

```text
<|speaker:1|>[surprised][excited]啊？盒子里真的有星星？
<|speaker:1|>【excited】啊？盒子里真的有星星？
```

## 官方语法摘要

| 用途 | 语法 | 建议 |
|---|---|---|
| S2 / S2.1 情绪、语气、表演方向 | `[自然语言描述]` | 主力用法，可写中文或英文，可放句首或句中 |
| 多说话人切换 | `<|speaker:0|>` / `<|speaker:1|>` | 爸爸/孩子这类对话必须用，配 `reference_id` 数组 |
| S1 旧模型情绪 | `(happy)` / `(sad)` / `(whispering)` | 只用于 S1；S2 API 直接用方括号 |
| 细粒度停顿/拟声实验 | `(break)` / `(long-break)` / `(breath)` / `(laugh)` / `(sigh)` | 可少量实验，但属于更不稳定的口语效果控制 |
| 普通口语节奏 | `嗯`、`啊`、`诶`、`咦`、`……`、`——` | 最稳定，应该直接写进台词 |

官方要点：

- S2 / S2.1 使用 `[bracket]` 方括号，不限固定标签集。
- tag 可以写自然语言描述，例如 `[whispers sweetly]`，中文也可写 `[低声说]`。
- tag 可以放句首，也可以放句中；放在哪里，就从哪里开始影响后面的词。
- 一个句子建议一个主情绪；复杂组合最多三种，不要每句堆标签。
- `temperature` 控制表现力；越高越有变化，越低越稳定。当前测试用 `1.0` 最容易听出差异，成稿可在 `0.85-1.0` 间试。

项目内落地规则比官方更保守：**一个句子只用一个主标签**，标签后留空格；除非是 `[whispering]`、`[sighing]` 这类短局部效果，否则不要多标签叠加。

项目内 TTS 路径也比官方示例更保守：`s2.1-pro-free` 只用于单声线或逐句拼接；真正的数组多说话人一次生成，必须等 `s2-pro` / `s2.1-pro` 稳定可用后再短测确认。

## 成稿推荐格式

```text
<|speaker:0|>[温和，带一点困惑，像在把球递给孩子] 我昨天看到一条新闻，第一眼就卡住了。
<|speaker:1|>[好奇，声音抬起来] 什么新闻？
<|speaker:0|>它说，美国有一片树林，四万七千多根树干，科学家却说它其实是一棵树。
<|speaker:1|>[惊讶，短促反问] 啊？一片树林，是一棵树？
<|speaker:0|>[放慢一点，像自己也在重新想] 对，我也卡在这儿。它凭什么算“一棵”？
```

## 标签怎么写更像对话

### 1. 写“说话状态”，少写抽象情绪

优先：

```text
[有点不敢相信，声音上扬]
[边想边说，语速慢一点]
[压低声音，像在讲秘密]
[突然想到了，语速快一点]
[有点担心，声音往下落]
```

谨慎：

```text
[curious]
[surprised]
[sad]
```

单个情绪词可以用，但对播客不够精确。对话需要的是“这个人为什么这样说、怎么说”。

主情绪建议直接写中文；英文短标签留给局部效果：

```text
主情绪：[试探着猜，像还没完全确定]
局部效果：我打开盒子，[gasping] 里面真的有东西。
```

### 2. 让标签服务台词，不替代台词

差：

```text
<|speaker:1|>[非常惊讶] 这是什么？
```

好：

```text
<|speaker:1|>[惊讶，先愣一下再反问] 啊？四万七千根……怎么会是一棵？
```

真正的听感来自“啊？”、“复读离谱词”、“省略号停顿”、“反问”，标签只是帮模型把这些读出来。

### 3. 标签短而具体

建议长度：6-18 个汉字，最多一个短逗号组合。标签后必须留一个 ASCII 空格。

```text
[低声说，像怕别人听见] 你先别告诉别人。
[兴奋，语速稍快] 快打开快打开！
[迟疑，边想边说] 除非……它们底下是连着的？
[担心，声音放轻] 那后面不就没人接上了吗？
```

避免很长的导演说明：

```text
[此处孩子处在一种复杂的心理状态中，既有震惊也有好奇，同时还带着对爸爸说法的不信任和探索欲]
```

这种会打断文本，也不利于稳定复现。

## 口语化手段优先级

| 优先级 | 手段 | 例子 | 说明 |
|---|---|---|---|
| P0 | 原始台词口语化 | `啊？`、`不是不是`、`等一下`、`我没转过来` | 最稳定，模型一定会读 |
| P0 | 标点节奏 | `……`、`——`、`？！` | 控停顿、拖长、反问、峰值 |
| P1 | 方括号表演提示 | `[边想边说]`、`[压低声音]` | 控读法，适合句首或变化点 |
| P1 | 多说话人整段生成 | `<|speaker:0|>` / `<|speaker:1|>` | 比逐句拼接更像真实对话 |
| P2 | 口语填充词 | `嗯`、`啊`、`诶`、`咦`、`哈` | 少量点睛，不要形成口癖 |
| P2 | 拟声/停顿效果 | `[sigh]`、`[gasp]`、`(break)` | 适合测试后小量使用 |

## 父子播客的标签风格

### 爸爸

爸爸不是老师，是一起探索的人。标签要偏“温和、困惑、递球、放慢、压低”，少用“权威、讲课、激昂”。

可复用标签：

```text
[温和，像自己也在想]
[放慢一点，把问题递出去]
[轻一点，像突然意识到问题不简单]
[压低声音，像讲一个反常识发现]
[带点笑意，但不夸张]
```

示例：

```text
<|speaker:0|>[温和，像自己也在想] 我第一次看到这句话，也没转过来。
<|speaker:0|>[放慢一点，把问题递出去] 如果地上是四万七千根树干，那“一棵树”到底藏在哪儿？
```

### 孩子

孩子承担更多起伏：惊讶、追问、猜测、突然想到、担心。标签可以更活，但仍要靠台词本身。

可复用标签：

```text
[一下来精神，声音抬起来]
[惊讶，短促反问]
[试探着猜，尾音上扬]
[突然想到了，语速快一点]
[担心，声音放轻]
```

示例：

```text
<|speaker:1|>[惊讶，短促反问] 啊？一片树林，是一棵树？
<|speaker:1|>[试探着猜，尾音上扬] 除非……它们底下是连着的？
<|speaker:1|>[突然想到了，语速快一点] 根呗！就像手指头分开，可底下连着一个手掌！
```

## 常用标签库

### 好奇 / 试探

```text
[好奇，尾音微微上扬]
[试探着猜，像还没完全确定]
[边想边说，语速慢一点]
[轻轻反问，像在确认]
```

### 惊讶 / 反常识

```text
[惊讶，先愣一下再说]
[不敢相信，声音上扬]
[短促惊呼，随后追问]
[突然被这个数字砸到]
```

### 恍然 / 想通

```text
[慢慢想明白，声音放平]
[恍然大悟，后半句变轻快]
[像拼图对上了，语气亮一点]
```

### 担心 / 难过

```text
[担心，声音放轻]
[慢下来，像看到后果]
[难过但克制，尾音落下去]
[小声说，像突然不太轻松]
```

### 讲秘密 / 反转钩

```text
[压低声音，像讲秘密]
[卖个关子，前半句放慢]
[轻一点，像要说重点]
```

### 轻松 / 带笑

```text
[带点笑意，不夸张]
[轻松一点，像随口打趣]
[笑着说，但别真的大笑]
```

## 句中标签用法

句中标签适合“读到某个信息时突然变调”：

```text
<|speaker:0|>这片林子看起来像四万七千棵树，[压低声音，强调反转] 但科学家说，它其实是一棵。
<|speaker:1|>我以为它只是老，[突然担心起来] 可如果没有小树干接上，那后面怎么办？
```

不要把句中标签插得太碎。每句最多一个变化点，除非是在做测试音频。

句中标签也要留空格：

```text
推荐：我打开盒子，[surprised] 里面真的有一颗小星星！
不推荐：我打开盒子，[surprised]里面真的有一颗小星星！
```

## 圆括号怎么用

对当前 S2/S2.1 成稿，不推荐用圆括号表达情绪：

```text
(happy) ...
(sad) ...
(whispering) ...
```

这些属于 S1 旧模型固定标签语法。S2 应改成：

```text
[happy] ...
[sad] ...
[whispering] ...
```

或更推荐：

```text
[带点笑意，不夸张] ...
[难过但克制，尾音落下去] ...
[低声说，像怕别人听见] ...
```

但 fine-grained control 里还有一组圆括号特殊效果，可以少量实验：

```text
(break)
(long-break)
(breath)
(laugh)
(cough)
(lip-smacking)
(sigh)
```

这类更像“舞台音效/口腔动作/停顿”，不等同于情绪。官方也提示部分效果仍在发展中，可能需要重复才明显。对正式播客，优先用 `……`、自然停顿词和 `[低声说]` 这类 S2 方括号。

## 成稿检查清单

- 是否每一轮对话都有明确说话人：`<|speaker:0|>` / `<|speaker:1|>`。
- 是否只在关键句加标签，而不是每句都加。
- 爸爸标签是否偏“温和、困惑、递球”，避免老师腔。
- 孩子标签是否承担起“惊讶、猜测、追问、担心”的起伏。
- 标签是否具体描述声音状态，而不只是抽象情绪词。
- 台词本身是否已经口语化；去掉标签后仍然像真人对话。
- 强声学效果是否可控：`[gasp]`、`[laughing]`、`[sigh]`、`[whispering]`、`[shouting]` 只在必要处用。
- 整段生成时是否保留上下文，不要把每句拆成独立 TTS 再拼。

## 推荐模板

```text
<|speaker:0|>[温和，像自己也没完全想明白] 我昨天看到一条新闻，第一眼就卡住了。
<|speaker:1|>[一下来精神，带着笑意] 啊？还有你看不明白的？
<|speaker:0|>嗯……不是看不明白，是这个说法太怪了。(break) 它说，一整片树林，其实是一棵树。
<|speaker:1|>[惊讶，声音上扬] 一整片树林？是一棵树？这不对吧！
<|speaker:0|>[放慢一点，把问题递出去] 对，所以咱们先别急着反驳。它凭什么敢这么说？
<|speaker:1|>[试探着猜，尾音上扬] 除非……它们底下是连着的？
```
