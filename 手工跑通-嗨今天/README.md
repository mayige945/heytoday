# 手工跑通-嗨今天目录索引

这里收纳“已经进入手工验证，但还没有资格写成正式系统”的流程材料。

它是 `探索/` 和正式项目之间的中间层：`探索/` 可以发散，`手工跑通-嗨今天/` 必须能按步骤执行，正式项目只吸收被多次验证过的结论。

## 目录规则

- 每个编号子目录就是一个手工任务。
- 每个子目录必须有 `README.md`，写清：目标、输入、步骤、产物、通过标准、沉淀位置。
- 这里放 runbook、短测方法、评估表、阶段结论；正式产品决策仍回到 `PRD_v0.8.md`、`Writing_Kit.md`、`prompts/` 或 `tts/`。
- 稿件正文继续放 `稿子/`；生成音频和请求记录继续放 `音频/`；这里不搬大产物，只做索引和流程控制。
- 一个手工任务连续跑通 3 次，并且没有靠临场判断救场，才考虑沉淀到正式 prompt、脚本或 PRD。

### 一期音频播客的实际流程

1. **新闻采集（独立工程模块，可选先跑）**：人工运行根目录 `新闻采集/` 的 CLI，得到分别适配小学高年级与初中的合格新闻池；模块不自动推荐或选题。
2. **选题与正文（继续手工）**：从合格新闻池或人工材料中给家长 3 个来源、题材多样的候选，标 1 个推荐；家长选题后，按当前小学高年级 prompt 写单主题正文，完成事实、风险和产品宪法自检。
3. **音频导演**：把正文改成口语对话，设计情绪、入口 / 证据 / 出口，标出多音字、专名、外文名和数字读法；选 6–8 个高风险 turn。
4. **MiniMax 生成**：先短测高风险 turn，确认声音区分、停顿、错读和控制词；通过后生成全篇，并核对 request 文本 / JSON、发音词典、mp3、时长和 `line_parts`。
5. **SenseVoice 预检与成人复听**：运行 `tts/评估音频.py`，生成 ASR / 情绪原始记录、复听排序和局部小修建议。所有高风险 turn、开头、结尾和至少 2 个随机 turn 必须成人复听；自动结果只负责路由，不能单独决定通过。自然度 MOS 尚未接入，仍使用固定成人复听表判断“像真人 / 像对话”。
6. **小修、发布与沉淀**：小修只改导演稿、发音词典、停顿、标签或局部重生成，最多 2–3 轮；通过后整理标题、描述、音频、来源和复听状态。封面先回到本期**新闻出处**选取直接相关的图片，保留原图、新闻页 URL、图片来源 / 署名与授权状态，再加工成 **1:1 方形播客封面**；若某个播放器另需横版，再额外出 16:9 版本。孩子反馈进入 `孩子反应日志.md`；稳定经验才进入 prompt、脚本或正式文档。
