# 04 发布准备

## 目标

把已经生成并初步核对的单期内容，整理成一组可播放、可分享、可给家长 review 的素材包。这个阶段不改新闻事实和正文结构；只检查对外呈现是否完整、准确、不过度承诺。

## 输入

- `稿子/日期_话题_小学档.md`
- `稿子/日期_话题_minimax导演稿_v*.md`
- `音频/日期_话题_minimax导演稿_v*.minimax.mp3`
- `音频/日期_话题_minimax导演稿_v*.minimax.minimax-request.txt`
- `音频/日期_话题_minimax导演稿_v*.minimax.minimax-request.json`
- 本期新闻出处页，以及从该页找到的直接相关图片
- 播放封面图片
- 本期成人复听或抽听结论

## 步骤

1. 确认音频文件存在，文件大小和时长合理；若未完整人工复听，必须标明“未完整复听”。
2. 从稿子元数据取 `内容描述（给平台/播放器）`，确认它能提示打开欲望，但不标题党、不提前收口、不外显尚未稳定的“爸爸和孩子/父子/儿子”角色关系。
3. **从新闻出处找图并留痕**：回到已选新闻的原始出处页，优先选与本期主问题直接相关的图片；非破坏性保存来源原图，并记录新闻页 URL、图片直链（若有）、图注 / 署名、图片提供方和可见的授权状态。找不到清楚来源或授权状态时，明确标“待核验”，不能假装可公开使用。
4. **加工播客封面**：默认产出 **1:1 方形封面**，缩略图里必须一眼看出本期主物件 / 画面，并和正文主题一致。可基于来源图做裁切、调色或非误导性的视觉加工，但不能篡改关键新闻事实、伪造新闻现场、抹去来源标识来规避授权；来源图有水印或授权不明时，只能作内部测试参考，公开前须取得授权或换成可用素材。某个播放器明确需要横版时，再额外生成 16:9 兄弟版本。
5. 准备对外标题、平台描述、音频路径、封面路径、来源说明、图片授权状态和风险提示。
6. 检查本期是否有热更新风险；如新闻赛果、时间、人物状态已经变化，先回到 `01_选题与初稿/` 更新稿子，再决定是否重出音频。
7. 把家长/成人复听结论和孩子反馈入口写清楚；孩子真实反馈进入 `孩子反应日志.md`。

## 产物

- 单期可播放素材清单，至少包括：
  - 标题
  - 平台/播放器内容描述
  - 音频文件路径
  - 播放封面路径
  - 新闻出处、封面原图路径 / URL、图片提供方与授权状态
  - 成人复听状态
  - 已知风险或待 review 点

## 通过标准

- 家长能一眼知道本期讲什么、为什么值得打开。
- 音频文件、1:1 封面、描述、新闻来源和图片来源说明齐全，路径可追溯。
- 封面主题和本期新闻一致；来源原图、加工版和图片授权状态都能对上。授权待核验的封面不得标为可公开发布。
- 平台描述和封面没有过度承诺当前尚未稳定的声音角色。
- 未完整复听、未跑 ASR、热更新风险等限制被明确标出。
- 没有把内部 review 信息、北极星词、制作指令或 TTS 控制词暴露给普通听者。

## 沉淀位置

- 稳定的发布素材格式写回 `prompts/` 或本目录。
- 封面生成和尺寸规则若反复使用，可新增到 `手工跑通-嗨今天/04_发布准备/` 的专门 playbook。
- 涉及产品定位或正式对外口径的变化，回到 `PRD_v0.9.md` 或 `Writing_Kit.md` 判断是否需要更新。
