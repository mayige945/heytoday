# 情绪 playbook v1 —— 通用版：不靠标签，把细腻情绪写进台词

**日期**：2026-07-01  
**状态**：正式通用 playbook  
**适用范围**：音频导演稿的通用情绪写法；适用于 MiniMax、Fish、MOSS 等 TTS 后端。  

> 来源说明：本 playbook 最初来自 workflow `emotion-via-wording-playbook`。Fish/MOSS 的实测结论是：很多“内在情绪”标签（如 `[surprised]`、`[curious]`）并不能稳定产生对应听感，真正稳的是台词语义、句式、标点、停顿和稀疏语气词。这个结论对当前 MiniMax 主线同样成立：标签和参数只能辅助，不能替代“这句话为什么有情绪”。

## 一句话策略
情绪长在台词语义里（猜对了/被砸到/替它揪心）+ 句式（复读反问·自我打断·追问链·卖关子）+ 标点（？ ？！ …… ——）+ 稀疏轮换语气词；TTS 后端的 emotion、native tag、韵律参数只做少量辅助。

## 写稿硬规则
1. 情绪必须长在台词语义里，不靠外挂。先定这句的"情绪由头"（猜对了/被真相砸到/替它揪着/被逗到）；由头到位，平实标点也读得出；由头没有，堆"哎呀"也是空的——禁用语气词假装情绪。
2. 惊讶/好奇/试探用"复读式反问"：把对方最离谱的词原样抠出来反问回去（"四万七千根？怎么会是一棵？"）。句式本身携带惊讶，不依赖标签。
3. 强标点（？！ ！ …… ——）只在每段一两处真正峰值打，其余留中性句做对比。全篇都打 = 被整体摊平。一句别堆三种以上强标点。
4. 显式停顿写出来：句中"……"=边想边说的迟疑；句首"……"=从低慢起点进入（担心/难过）；"——"=拖长前一个音或标记自我打断转折。
5. 语气词稀疏、轮换、只点睛：诶/咦/哦/哇/嗯/哈/嘿轮用，绝不每句上、绝不固定成一个口头禅（撞 B-4 黑名单）。情绪主力交给句式扛。
6. 一冷一热分工：情绪起伏几乎全压孩子（多用？！、感叹语气词、短促叠句）；爸爸≈中性偏暖、稳，多句号和试探性问号——既突出对比，又守"爸爸问句多于结论、是玩伴不是老师"。
7. 正反馈落在"孩子自己那句叹号"上，爸爸绝不接一句夸奖收（B-4 捧场缴械族禁）。爸爸表达暖意用示弱代替表扬（"你这么一捋，我那没读通的劲儿松了点"）。
8. 担心/难过只点到"后果与画面"即止，句首省略号留白往下沉、不反弹，结尾留气不用问号上挑，绝不滑向"所以我们要保护资源"。
9. 俏皮只用于"卖关子/装傻把球踢回孩子"，靠破折号"先吊后抖"的节奏，不用于爸爸抖机灵显摆。
10. 逗号疏密调急缓：多逗号=切碎放慢揪着说（担心）；少逗号连成长句=一口气冲出来（惊喜）。

## 情绪→配方→示例
| 情绪 | 配方 | 示例 |
|---|---|---|
| 惊讶 | 单字感叹独立成拍（啊？/诶？）+ 复读反问最离谱的词；短促？！收。爸爸版用自我打断"等等，你慢点——你是说……" | 孩子：啊？四万七千根……怎么会是一棵？ |
| 好奇 | 咦/诶 + 半截假设（是不是…/除非…）句尾省略号留白；招牌"追问链"；降调试探放慢 | 孩子：四万七千棵……除非，它们底下是连着的？ |
| 惊喜 | 对！/哈，我知道了—— + 短促肯定句、！收得上扬；"不是A，是B！"揭晓；落孩子自己的发现，爸爸不接夸 | 孩子：根呗！五根手指头分开，底下连着一个手掌——它们其实是……一棵！ |
| 恍然 | 拖长"哦——/啊……"起手 + 先长省略号留白再慢吐结论；先降后平 | 孩子：哦……一个人嘛，当然一块儿黄。 |
| 担心 | 句首省略号/唉哎降调起 + 短句后果推演不反弹；结尾省略号留气；多逗号切碎 | 孩子：……那等这些老树干一根一根倒下去，后面，就没人接上了。 |
| 温暖 | 爸爸底色：平缓短句 + 嗯/来/好低能量收束、句尾句号；示弱代替夸奖 | 爸爸：……你这么一捋，我那没读通的劲儿，松了点。 |
| 俏皮 | 哈/嘿/哎哟 + 卖关子（破折号先吊后抖）或装傻把球踢回；短句轻快 | 爸爸：说出来你八成不信——四万七千多棵树，其实是一棵。 |

## 后端使用原则

1. **先修文本，再调参数**：如果一句话听起来没有惊讶、担心、恍然，优先改情绪由头、句式和停顿，不先堆标签。
2. **标签少量点睛**：native sound tag、emotion、voice_modify、行内标签都只能用于少数关键 turn；全篇依赖标签会变演、变吵或造成角色不稳。
3. **短样验证优先**：任何新标签、新参数、新音色变化，先进 20-40 秒短样，不直接进整期。
4. **角色稳定优先于情绪强度**：爸爸/孩子声音稳定比“更有戏”重要；造成变声、串声、夸张表演的控制项，正式稿禁用或降级。
5. **当前 MiniMax 主线**：优先用台词、标点、`<#0.30#>` 等停顿、少量 `（温和）（慢）（快）`；`voice_modify` 和强情绪标签只做边界测试，不进常规整期。

## 后端附录：Fish 韵律参数（历史实现）
`temperature=0.85`（最该动的旋钮，官方"controls expressiveness"，从默认0.7上调；读数字/专名嫌乱可降到0.6）、`top_p=0.7`（不动，只用 temperature 一个旋钮）、`prosody.speed=1.05–1.10`（活泼底色，别超1.15）、`volume=0`、`chunk_length=300` + `condition_on_previous_chunks=true`（跨段续语气、防摊平割裂）、`normalize=true`（数字/中英混读必开）。

## 后端附录：Fish 标签政策（历史实现）
Fish 历史测试中只留 4 个有声学特征、实测生效的：`[laughing]`（孩子一声真笑）、`[whispering]`（爸爸压低卖关子，配 speed≈0.92）、`[shouting]`（极少，真惊呼，慎用）、`[crying loudly]`（几乎不用）。其余 [surprised][curious][worried][excited][sad][sighing][break] 全删——实测听不出。用法：官方英文小写、放句首、与中文间留空格、一句最多一个、整段≤3、能不用就不用。

## 参考音方案（★最大杠杆，需你动手）
**原理：克隆只复刻它听过的东西，平样本必然平输出。** 你现在两个锚点是从 VoxCPM 平合成锚点克隆来的，天生没情绪范围——这是标签推不动的根因。
**做法**：爸爸+孩子各真人录一条 15–40 秒、**带情绪起伏**的参考音（想要兴奋就真演兴奋，平语气假装=只学到平的）。
- 爸爸（温和、问句多于结论）："哎你看这个——真的假的？这也太有意思了吧。嗯……我也不太确定为什么，咱俩一起想想看。不过说真的，这事儿还挺好玩的，你说呢？"
- 孩子（清亮、好奇、活泼）："哇——真的吗？！那到底是为什么呀？我有点想不明白……嘿嘿，爸爸你是不是也不知道？哎我突然想到一个事儿！"
- 录音要点：安静房间、手机怼近约20cm、一次连贯读完取最自然一遍、括号情绪要真演、无回声无背景音（一处回声毁整条）、≥24kHz/16bit、句间留1–1.5秒。
- 现成 `tts/voices/raw/孩子.wav`（87秒真人小孩）可剪 20–30 秒情绪最丰富一段先顶替试。
- 落地：①各得带情绪 wav → ②转单声道≥24kHz、normalize → ③弃用旧 *_anchor.wav/*_synth.wav → ④Fish 控制台重传建 voice model 拿新 id 填进 .env。这条参考音 MOSS 接入时直接当 PROMPT_AUDIO_SPEAKERn，通用。

## 后端附录：迁移到 MOSS
MOSS-TTSD **没有任何情绪控制项/标签**，情绪 100% 靠上下文+台词措辞+标点（官方示例 `[S1] Promising. There it is. I hate that word.` 全靠短句断句与重复，零标签）。所以这套"情绪写进措辞+标点"对 MOSS 不仅能迁移，本来就是唯一有效载体，原样复用；Fish 那 4 个行内标签在 MOSS 上要全部去掉、改写进台词。MOSS 可调的只有采样参数（temperature≈1.1/top_p0.9/rep_penalty1.1），那是控随机性不是控情绪。

> 通用结论：一条情绪有由头的台词，比十个情绪标签更可靠。
