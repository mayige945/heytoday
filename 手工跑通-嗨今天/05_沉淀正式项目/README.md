# 05 沉淀正式项目

## 目标

把连续手工跑通的经验，转成正式项目资产，而不是永久停留在调研文档里。

## 输入

- 前四步的产物和复盘记录
- `孩子反应日志.md`
- `音频/*.request.json`
- 多期可比对的导演稿和成品音频

## 步骤

1. 只挑“重复出现、可描述、可执行”的经验沉淀。
2. 判断它应该进入哪一层：prompt、脚本、PRD、Writing Kit、内容安全政策，还是继续留在手工跑通。
3. 对照根目录 `正式文档索引.md`，确认下游指针、README、AGENTS/CLAUDE 是否需要同步。
4. 更新目标文件时写明版本、日期、为什么改。
5. 把被替代的旧实验材料移到 `99_历史探测/`，或在旧版顶部标明“已归档；当前运行版见 ...”。

## 通过标准

- 新规则能减少下一期手工判断，而不是增加散文说明。
- 新规则不违反 PRD 产品宪法。
- 新规则有至少 3 次真实样本支撑，或者明确标为临时假设。
- `正式文档索引.md` 已更新当前指针；没有留下会误导下一轮执行的旧版本入口。

## 沉淀位置

- 写稿规则：`prompts/`，必要时同步 `Writing_Kit.md`。
- TTS 参数和后端：`tts/README.md`、`tts/出音频.py`。
- 产品决策：`PRD_v0.9.md`。
- 文档导航和收尾检查：`正式文档索引.md`。
- 仍不稳定的嗨今天主线材料：继续留在 `手工跑通-嗨今天/`；奥特曼定制资讯材料继续留在 `奥特曼咨询番外/`。
