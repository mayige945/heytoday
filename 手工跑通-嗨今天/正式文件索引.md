## 维护规则

1. 新增正式文件时，先确认它不是单期过程记录，再加入“正式文件”表。
2. 对正式文件，必要时在文件头部补 `状态`、`适用范围`、`当前/归档`。
3. 若某个正式文件被替代，旧版顶部标“已归档；当前运行版见 ...”，并移入“不进入正式文件集合”或改为历史材料。
4. 保持 `手工跑通-嗨今天/README.md` 仍只做嗨今天主线的一级目录索引，不把本表内容塞回目录索引。奥特曼咨询番外作为项目根目录的平级入口，由其自身 README 维护。
