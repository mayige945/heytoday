**日期**：2026-07-04  
**状态**：MVP 手工跑通阶段指导书  
**适用范围**：「喂，今天」小学高年级档，爸爸 + 孩子双人新闻播客  

---

## 更新记录

### v0.1 - 2026-07-04

围绕“先把播客稿交给 LLM 音频导演，再用 MiniMax 生成音频”的路线，沉淀第一版研究与手工操作方法。

复核修订：

- 复核 MiniMax endpoint：`.io` 为国际版 endpoint；本项目当前 key 属于 `.com` 国内版，正式手工主线默认使用 `https://api.minimaxi.com/v1/t2a_v2`。
- 把 `voice_setting.emotion` 从旧测试里的 `neutral` 修正为官方枚举 `calm`，并明确 Speech 2.8 主流程不使用 `whisper`。
- 修正系统音色列表链接到 `/docs/faq/system-voice-id`。

关键结论：

1. MiniMax Speech 2.8 适合做“口语化、带停顿、带语气词、参数可控”的播客音频。
2. 但截至本次核查，MiniMax 公开 T2A HTTP / WebSocket 文档里没有看到类似 Fish `reference_id[] + <|speaker:n|>` 的原生多说话人对话协议。
3. 因此正式路线应改成：**先探测 MiniMax 原生多说话人是否可用；若不可用，再使用 `line_concat` 逐句/逐 turn 拼接作为可控基线**。
4. 音频成败不能只看“像不像 TTS”，要看孩子是否愿意听、是否被对话勾住，以及爸爸是否仍像一起探索的人。

补充短测结论（2026-07-06）：

- E 组“稳定音色轻情绪”经复听认可，暂定为正式导演稿首选强度。
- D 组“强控制上限”出现同一角色前后变音，正式稿禁用大幅 `voice_modify` 和跨 turn 音色微调。
- `request.txt` 只看实际朗读文本；情绪、语速、音高、音色微调必须查 `request.json`。
- `.io` 是国际版 endpoint，当前 `.com` key 不通用；手工主线默认只使用 `https://api.minimaxi.com/v1/t2a_v2`。脚本仍支持通过 `MINIMAX_ENDPOINT` / `MINIMAX_ENDPOINTS` 显式配置其它 endpoint，并在 request JSON 记录最终使用的 endpoint。

## 1. 目标

当前不是做自动化系统，而是把手工 MVP 的第三步固定下来：

```text
无情绪初稿
  -> LLM 音频导演稿
  -> MiniMax 短样验证
  -> 调整控制词 / 参数 / 停顿
  -> 生成整期
  -> 成人预听 + 孩子反应记录
  -> 回流到音频导演规则
```

本环节只服务一个目标：让一篇已经合格的「喂，今天」文字稿，变成早餐桌上能听下去、像父子一起探索的播客音频。

不做：

- 不搭 App / RSS / 新闻流水线。
- 不把 TTS 质量当产品北极星。
- 不为了展示技术而牺牲“爸爸不是老师、问题大于答案、首先要有趣”。
- 不默认 MiniMax 有原生双人对话，必须先验证。

---

## 2. 项目约束

来自 PRD / Writing Kit / 当前 prompt 的硬约束：

- 听众：小学高年级，约 9-12 岁。
- 形式：固定爸爸 + 孩子两个声音。
- 场景感：早餐桌，但不必每句都模拟吃饭。
- 一期只聊一条主题新闻。
- 爸爸不是老师，是一起探索的人。
- 爸爸的问句要多于结论。
- 教育必须隐身，禁止“今天告诉我们”“所以我们要”“小朋友应该”。
- 结尾留开放问题，不给标准答案。

音频层的约束：

- 爸爸声音稳定、温和、好奇、略慢，不要主持人腔和老师腔。
- 孩子声音更活，但不能假可爱、不能一直尖叫。
- 情绪起伏来自“发现过程”，不是每句贴强情绪标签。
- 换人节奏要像聊天，不能像两个孤立音频文件硬拼。

## 3. 类似赛道观察

本节不是找直接竞品，而是拆“儿童新闻/知识播客”在音频上的可借鉴点和风险。

| 样本 | 可借鉴点 | 反面风险 | 对「喂，今天」的启发 |
|---|---|---|---|
| KidNuz | 短、清楚、节奏快，有 quiz 机制，适合每天听 | 多条新闻快报天然偏“知道信息”，深聊空间不足 | 借它的清晰开场和节奏，不借多条速览 |
| The Ten News | 10 分钟左右，有儿童新闻节目的能量和结构 | 容易变成栏目感、主持人感，孩子像听节目而不是参与思考 | 借“短时段内有起伏”，但爸爸不能变主持人 |
| ABC Kids Listen News Time | 善用声音、问题、故事化讲新闻 | 生产感强，可能离家庭早餐桌太远 | 借音频画面感，不做花哨声效 |
| 小新聞動動腦 | 中文儿童新闻语境更近，也强调“动脑” | 若处理不好，容易显性教育化 | 可借“新闻 + 想一想”，但北极星词不能直接出场 |
| 少年参考 | 中文青少年新闻解释有密度 | 年龄偏高，容易变成成人新闻降级版 | 可借选题视野，不借讲解腔 |
| Brains On | 孩子声音参与、问题驱动、好奇心强 | 多角色、棚内制作感强，不适合当前手工低成本 | 借“孩子真的在问”，不是让孩子只负责捧哏 |

结论：

- 正面：好儿童音频都有“清楚的钩子、稳定角色、强弱起伏、可预期但不死板的结构”。
- 反面：儿童新闻很容易滑向“快报 + quiz + 主持人总结”，这和本项目“单主题深聊、不收口、不说教”冲突。
- 对 MiniMax 的要求：不是生成“新闻播报”，而是生成**低制作感、强对话感、轻情绪曲线**。

## 4. MiniMax 能力地图

### 4.1 已确认适合使用的能力

根据 MiniMax 官方 T2A 文档、Speech 2.8 发布说明，以及本仓 `tts/出音频.py` 已跑过的控制面板测试，MiniMax 当前可作为“精细可控单声线 TTS”使用。

可用控制项：

- HTTP endpoint：本项目当前使用 MiniMax 国内版 `.com` key，默认 endpoint 为 `https://api.minimaxi.com/v1/t2a_v2`。国际版 `.io` endpoint 不与当前 key 混用；如更换国际版 key，再显式配置 `MINIMAX_ENDPOINT`。
- `text`：同步 HTTP 小于 10,000 字符；超过 3,000 字符官方建议用 streaming；异步长文本可走 async，官方 async guide 标注文本输入上限可到 1M 字符。
- `\n`：段落换气。
- `<#x#>`：停顿控制，官方范围 `[0.01, 99.99]` 秒。
- Speech 2.8 native sound tags：如 `(laughs)`、`(chuckle)`、`(breath)`、`(gasps)`、`(sighs)`、`(emm)` 等。
- inline pronunciation：可用拼音、IPA、粤拼修正读音。
- `pronunciation_dict`：适合固定专名，如 `Pando/(pan1)(duo1)`。
- `voice_setting.voice_id`：系统音色或复刻音色。
- `voice_setting.speed / vol / pitch`：语速、音量、音高。
- `voice_setting.emotion`：官方枚举为 `happy / sad / angry / fearful / disgusted / surprised / calm / fluent / whisper`；本仓 Speech 2.8 主流程只采用 `happy / sad / angry / fearful / disgusted / surprised / calm`，不使用 `whisper`。
- `voice_modify.pitch / intensity / timbre`：整体音色微调，只用于边界测试，不进入正式稿。
- `voice_modify.sound_effects`：如 `spacious_echo`、`lofi_telephone`、`robotic`，正式播客原则上不用，只做测试。
- `subtitle_enable / subtitle_type`：可用于后续核对句级或词级时间戳。

### 4.2 关键未知：原生多说话人对话

截至 2026-07-04，本次核查的公开资料中：

- T2A HTTP 示例是一个 `text` + 一个 `voice_setting.voice_id`。
- T2A WebSocket 是 `task_start` 设置一个 `voice_setting`，`task_continue` 继续传文本。
- 官方页面未显示 Fish 那类 `<|speaker:0|>` / `<|speaker:1|>` 多说话人标签。
- 本地扫描 MiniMax OpenAPI 文档时，未找到 `speaker` / `dialogue` 字段。
- `timbre_weights` 是 legacy 混合音色字段，不等于多角色对话；本仓旧测试也记录过系统音色下可能返回 `voice id not exist`。

因此，不能说 MiniMax 已经有“原生双人对话 API”。更准确的判断是：

> MiniMax 有很强的口语化、停顿、情绪和音色控制；是否支持公开 API 原生多说话人换声，需要单独探测。

### 4.3 路线优先级

1. **P0：MiniMax 原生多说话人探测**
   - 如果公开 API 或控制台确认支持多角色，并且短样稳定，优先走原生对话。
   - 这是理论上的最优路线，因为上下文、节奏、抢话感、换人呼吸都更自然。

2. **P1：MiniMax `line_concat` 可控基线**
   - 当前仓库已实现：每个 turn 用单个 voice_id 生成，再按同人/换人停顿拼接。
   - 优点：声音稳定、参数细、失败可定位。
   - 缺点：整段互动感弱于原生对话，换人边界可能太整齐。

3. **P2：Fish S2-Pro 原生对话**
   - 如果有 S2-Pro 稳定额度，Fish 仍是已知更接近“多说话人原生对话”的路径。
   - 当前问题是额度和模型可用性。

## 5. MiniMax 原生多说话人探测流程

探测目标：确认 MiniMax 是否能在一次请求里稳定输出爸爸/孩子两个不同声音，并保留整段对话上下文。

### 5.1 文档探测

每次重启 MiniMax 研究前，先查：

- T2A HTTP 文档是否出现 `speaker`、`dialogue`、`role`、`multi-speaker`、`voice_ids`、`reference_id[]`。
- T2A WebSocket 是否允许在同一 session 的不同 `task_continue` 中切换 `voice_setting`。
- async 长文本是否支持 JSON 结构化多角色输入。
- MiniMax Audio 网页产品是否有多角色功能，以及该功能是否开放 API。

判定：

- 找到官方字段和示例：进入 API 短测。
- 没找到：不要猜，继续用 `line_concat` 基线。

### 5.2 API 短测

只做 20-40 秒，避免烧成本。

候选探测：

1. 单请求文本里写 `爸爸：... 孩子：...`，看模型是否只读成同一 voice。
2. 单请求文本里尝试 `<|speaker:0|>` / `<|speaker:1|>`，看是否被读出来或报错。
3. WebSocket 同一 session 内尝试不同 `task_continue` 切换 voice 设置；如果协议不支持，不把它当正式路线。
4. 若找到官方多角色字段，用 6-8 个 turn 测声音稳定、角色不串、标签不读出。

通过标准：

- 爸爸和孩子声音可明显区分。
- 角色切换不串、不吞、不把标签念出来。
- 不是靠文本里的“爸爸说/孩子说”假装区分。
- 30 秒短样里至少 6 次换人仍稳定。

失败标准：

- 全段只有一个声音。
- “爸爸：”“孩子：”被念出来。
- speaker 标签被念出来。
- 第二个声音偶发丢失。
- 只能靠后期剪辑或手动分轨救。

## 6. 音频导演稿规范

### 6.1 文件形态

建议从已有初稿复制一份：

```text
稿子/日期_话题_minimax导演稿_v1.md
```

正文仍保持当前解析器可识别的格式：

```text
**爸爸**：（压低 慢）先别急，我给你看一件有点离谱的事。<#0.45#>

**孩子**：（惊 快）啊？一棵树，变成一片树林？
```

不要引入新 DSL。当前脚本已经能解析 `（舞台提示）`、`<#x#>`、`\n`。

### 6.2 导演原则

先写“为什么这样说”，再写“怎么读”。

正确顺序：

```text
这句在剧情里的作用 -> 台词是否自然 -> 标点/停顿 -> MiniMax 控制词
```

错误顺序：

```text
先堆 emotion / speed / pitch -> 再指望声音自己变成播客
```

### 6.3 角色分工

爸爸：

- 主色：温和、慢一点、压低、平静、带笑、轻声。
- 作用：递问题、卡住、示弱、收住。
- 避免：大声、兴奋演讲、权威讲解、老师总结。

孩子：

- 主色：好奇、惊讶、试探、突然想到、担心、轻快。
- 作用：制造情绪峰值、追问、猜测、接住反常识。
- 避免：每句都喊、每句都惊讶、假可爱。

### 6.4 控制词预算

一整期建议：

- 50%-60% 台词不加舞台提示，只靠文本、标点和 `<#x#>`。
- 30%-40% 使用轻提示：`（温和）`、`（慢）`、`（快）`。
- 0%-10% 使用明显情绪：`（惊）`、`（兴奋）`、`（紧张）`、`（叹气）`，先只放在短测里，整期谨慎。
- 正式稿默认不手写 `(gasps)` / `(laughs)` / `(sighs)`；确实需要时，一句只出现一次，且不要同时写中文同义舞台提示。

### 6.5 推荐提示词表

当前脚本能较好映射的中文舞台提示：

| 目标 | 写法 | 作用 |
|---|---|---|
| 平稳 | `（平静）` / `（温和）` | `emotion=calm`，少用，优先让台词自然变平 |
| 放慢 | `（慢）` | `speed` 降低，适合爸爸留问题、担心段 |
| 加快 | `（快）` / `（兴奋）` | `speed` 提高，适合孩子突然想到 |
| 惊讶 | 优先用 `？！`、复读、短句 | 正式稿先不用 `（惊）`，避免自动加入 `(gasps)` |
| 开心 | 优先用台词和标点 | 正式稿先不用 `（笑）` / `(laughs)` |
| 担心 | 优先用慢速、停顿、问题句 | `（紧张）` 只进短测，儿童稿慎用 |
| 下沉 | 优先用 `（慢）` + 停顿 | 正式稿先不用 `（叹气）` / `（低）` |
| 音色微调 | 不用于正式稿 | `voice_modify` 已验证会造成同一角色前后变音 |
| 停顿 | `<#0.30#>` / `<#0.60#>` | 精确停顿 |
| 换气 | `\n` | 段落换气 |

正式稿不建议使用：

- `（更低沉）` / `（更明亮）` / `（浑厚）` / `（清脆）` / `（柔和）` / `（有力）`
- 手写 `(gasps)` / `(laughs)` / `(sighs)`，除非单独短测通过
- `（音效=robotic）`
- `（音效=lofi_telephone）`
- `（音效=spacious_echo）`
- 过多 `（大声）` / `（喊）`
- 长篇复杂舞台说明

## 7. 情绪曲线模板

不是每期照抄，但 MiniMax 导演稿可以用这条曲线自检：

```text
轻松碰头
  -> 反常识钩子
  -> 爸爸卡住
  -> 孩子试探
  -> 孩子猜中/发现
  -> 情绪回落
  -> 新事实推进
  -> 轻微担心或复杂性浮出
  -> 再推理
  -> 小转机
  -> 更深问题
  -> 留白收束
```

能量分配：

- 开头 30 秒必须有“啊？”。
- 第一个峰值给孩子，不给爸爸。
- 中段必须有回落，否则全程会吵。
- 沉重话题只下沉一点，立刻给出口。
- 结尾声音要收住，不要爸爸总结。

## 8. 手工生成流程

### Step 0：选一篇基线稿

优先使用已有稿：

```text
稿子/2026-06-30_Pando一棵树的树林_小学档.md
```

原因：

- 已有平版音频。
- 已有 Fish 情绪编排版。
- 已有 MiniMax 控制面板和高情绪短测。
- 方便 A/B。

### Step 1：LLM 生成 MiniMax 音频导演稿

可直接给 LLM 的指令：

```text
你是「喂，今天」的音频导演。请把下面这篇无情绪初稿改成 MiniMax Speech-2.8 可生成的音频导演稿。

硬约束：
1. 保留 **爸爸**：/ **孩子**：格式。
2. 不改新闻事实，不新增教育总结。
3. 爸爸仍是一起探索的人，不是老师。
4. 情绪来自台词，不靠每句贴标签。
5. 正式稿优先只使用这些 MiniMax 提示：
   （温和）（平静）（慢）（快）
   如需惊讶、开心、担心、下沉，优先改台词、标点和停顿，不先加强情绪标签。
6. 可少量插入 <#0.30#>、<#0.45#>、<#0.60#>、<#0.80#> 停顿。
7. 可用 \n 表示换一口气。
8. 不要用 Fish 方括号标签。
9. 不要使用 `更低沉 / 浑厚 / 更明亮 / 清脆 / 柔和 / 有力` 这类会触发 `voice_modify` 的音色词。
10. 不要手写 `(gasps)/(laughs)/(sighs)`，除非本期短测证明不突兀。

输出：
- 一份可直接跑 tts/出音频.py 的 Markdown 正文。
- 最后列出“本期情绪曲线”和“最该短测的 6-8 个 turn”。
```

### Step 2：只截 20-40 秒短样

短样必须覆盖：

- 爸爸低能量开场。
- 孩子惊讶反问。
- 爸爸递问题。
- 孩子试探或猜中。
- 一处停顿。
- 一处轻量 `voice_setting` 控制，如 `（温和）`、`（慢）` 或 `（快）`；native sound tag 只作为边界测试，不作为正式短样必选项。

短样通过后再整期。不要先生成 8-10 分钟再听。

### Step 3：短样 A/B

建议顺序：

1. **A：无控制基线**  
   去掉所有括号提示，只保留口语台词。

2. **B：台词 + 标点 + 停顿**  
   加 `……`、`——`、`？！`、`<#0.45#>`，不加 emotion。

3. **C：轻 emotion**  
   只给关键 turn 加 `（压低）`、`（惊）`、`（兴奋）`、`（慢）`。

4. **D：强控制上限**  
   测 `pitch / vol / voice_modify / native tags` 的边界，听清哪些会过演。

5. **E：稳定音色轻情绪**
   不使用 `voice_modify`，不手写 native tags；只保留少量 `speed` / `emotion=calm`。这是当前正式整期的首选强度。

正式整期优先使用 E 强度；B 可作为更稳的退路，C 只在短测确认不过演时少量采用。D 只用于摸边界和反例留档。

### Step 4：整期生成

当前命令：

```powershell
$env:HEYTODAY_TTS_BACKEND="minimax"
D:/project-script/VoxCPM/.venv/Scripts/python.exe tts/出音频.py 稿子/日期_话题_minimax导演稿_v1.md
```

生成后检查：

- `音频/*.minimax-request.txt`：实际发给 MiniMax 的文本。
- `音频/*.minimax-request.json`：每句 voice_setting / voice_modify / text。
- `音频/*.line_parts/`：逐 turn 音频，方便定位坏句。

### Step 5：人工预听修正

优先修文本，不优先改参数。

常见问题：

| 问题 | 优先修法 |
|---|---|
| 太平 | 改台词句式，加反问/复读/停顿，再少量 emotion |
| 太演 | 删强标签，降低孩子的快/喊/高音 |
| 像老师 | 改爸爸台词，删总结句，增加“我卡住的是...” |
| 拼接感重 | 缩短换人停顿，合并同一说话人连续短 turn，减少每句结尾强停顿 |
| 孩子假可爱 | 降 pitch，删过多“哇/呀/嘿嘿” |
| 爸爸像主持人 | 降 speed，删节目腔开场，改成具体困惑 |

## 9. 评估方法

### 9.1 成人预听硬门槛

任一命中即不放给孩子：

- 标签被念出来。
- 爸爸/孩子声音明显串或角色错。
- 爸爸出现老师总结腔。
- 情绪过猛，像短视频配音。
- 关键新闻事实听不清。
- 音量忽大忽小影响理解。
- 整段超过孩子当前可承受长度。

### 9.2 成人评分表

每项 1-3 分，低于 20 分不进孩子测试。

| 维度 | 1 分 | 2 分 | 3 分 |
|---|---|---|---|
| 角色稳定 | 串声/错声 | 偶尔不稳 | 爸爸孩子清楚 |
| 对话感 | 像两段朗读 | 基本像聊天 | 有来回、有接住 |
| 爸爸状态 | 老师/主持人 | 偶有讲解腔 | 一起探索 |
| 孩子状态 | 假可爱/吵 | 有些表演 | 好奇自然 |
| 情绪曲线 | 平或乱 | 有起伏但生硬 | 起落自然 |
| 停顿节奏 | 赶/拖 | 基本可听 | 像真实聊天 |
| 信息清楚 | 听不清重点 | 大致清楚 | 事实与问题都清楚 |
| 不说教 | 明显收口 | 偶有微总结 | 完全隐身 |

### 9.3 孩子反应记录

沿用 `孩子反应日志.md`，音频环节额外加：

```text
- **音频听感反应**：
  - 有没有说“这个声音怪/假/吵/像机器人”：
  - 有没有模仿某句台词或声音：
  - 哪一段眼神亮起来：
  - 哪一段开始烦躁或走神：
  - 是否主动问“明天还有吗”：
```

判断：

- 完播 + 至少 1 次主动追问：音频可继续。
- 完播但无追问：先看稿子，不急着改 TTS。
- 未完播且抱怨声音：优先改音频。
- 未完播但没抱怨声音：优先看稿子节奏和内容。
