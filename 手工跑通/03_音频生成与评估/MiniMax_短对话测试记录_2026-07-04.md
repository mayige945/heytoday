## 待人工复听

优先复听顺序建议：E -> B -> C -> A -> D。

- E：当前最接近“正式可用强度”的候选，用于验证稳定音色下是否仍有对话感。
- B：如果只靠标点和停顿已经像聊天，正式稿优先用 B 强度。
- C：如果孩子惊讶自然、爸爸没有老师腔，正式稿可少量采用 C 强度。
- A：用于判断默认音色和 `line_concat` 拼接本身是否已经可接受。
- D：只看边界，重点检查是否过演、是否像短视频配音、native tags 是否被读出。

人工复听时按指导书硬门槛记录：角色是否稳定、控制词是否外露、停顿是否突兀、爸爸是否像一起探索的人、孩子是否自然。

## 人工复听反馈

### E 稳定音色轻情绪

用户复听反馈：E 没问题，可以继续。

结论：E 暂定为后续正式导演稿的首选控制强度。它保留少量 `speed` / `emotion=calm`，但不使用 `voice_modify`、不手写 native tags、不改 `pitch`，因此比 C 更稳，比 B 略有节奏控制。

### D 强控制上限

用户复听反馈：前后声音不一致，第一段开始就有变音感。

核对 request JSON 后确认，D 不是“没加情绪”，而是控制过重：

- 第 1 句爸爸：`speed=0.88`、`pitch=-4`，并叠加 `voice_modify.pitch=-45`、`voice_modify.timbre=-55`。
- 第 3 句爸爸：`emotion=happy`，并插入 `(laughs)`。
- 第 5 句爸爸：`pitch=-2`、`emotion=sad`，并叠加 `voice_modify.intensity=55`。
- 第 2 句孩子：中文 `（惊呼）` 自动加 `(gasps)`，原文又手写 `(gasps)`，实际请求里出现 `(gasps)(gasps)`。

结论：D 达到了“强控制上限”测试目的，但不能作为正式稿强度参考。正式路线应避免同一角色跨 turn 使用大幅 `voice_modify`，尤其是 `pitch/timbre`；优先使用 B 或 C 的控制强度。

后续规则：

- `request.txt` 只用于检查实际朗读文本、native tags 和停顿标签是否外露；情绪和音色参数必须看 `request.json`。
- 正式稿默认不使用 `更低沉 / 浑厚 / 更明亮 / 清脆 / 柔和 / 有力` 这类 `voice_modify` 控制。
- 同一角色每期应保持基础 `voice_id`、`pitch`、`timbre` 稳定；情绪主要靠台词、标点、停顿和少量 `emotion`。
- 不要同时写中文惊呼/笑/叹气舞台提示和手写原生 tag，避免 `(gasps)(gasps)` 这类重复。
