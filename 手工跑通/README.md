# 手工跑通目录索引

这里收纳“已经进入手工验证，但还没有资格写成正式系统”的流程材料。

它是 `探索/` 和正式项目之间的中间层：`探索/` 可以发散，`手工跑通/` 必须能按步骤执行，正式项目只吸收被多次验证过的结论。

## 目录规则

- 每个编号子目录就是一个手工任务。
- 每个子目录必须有 `README.md`，写清：目标、输入、步骤、产物、通过标准、沉淀位置。
- 这里放 runbook、短测方法、评估表、阶段结论；正式产品决策仍回到 `PRD_v0.6.md`、`Writing_Kit.md`、`prompts/` 或 `tts/`。
- 稿件正文继续放 `稿子/`；生成音频和请求记录继续放 `音频/`；这里不搬大产物，只做索引和流程控制。
- 一个手工任务连续跑通 3 次，并且没有靠临场判断救场，才考虑沉淀到正式 prompt、脚本或 PRD。

## 当前主线

正式文件候选清单见：`正式文件索引.md`。该清单用于确认哪些 runbook / 指导书 / playbook 进入正式文件集合，过程记录不进入。

| 目录 | 当前职责 | 什么时候进入 |
|---|---|---|
| `01_选题与初稿/` | 从候选新闻到无音频导演的正文初稿；候选必须保持来源/题材多样性，并说明孩子为什么会想听。 | 每天先选题、写正文时 |
| `02_音频导演稿/` | 把正文改成可 TTS 的口语化、带停顿、带轻情绪导演稿；导演前先标出多音字、专名、外文名、数字读法等发音风险。 | 正文通过后，需要出音频时 |
| `03_音频生成与评估/` | 按当前 MiniMax 主线生成音频，并完成短测、发音字典核对、自动预检、人工复听和小修。 | 导演稿通过后，开始生成和复听时 |
| `04_沉淀正式项目/` | 把稳定手工经验写回正式 prompt、脚本和项目文档。 | 某条经验重复出现、下一期还会用时 |
| `99_历史探测/` | 保留已探过但暂不作为主线的后端、标签、音色实验。 | 查旧实验结论或归档被替代材料时 |

本索引只维护 `手工跑通/` 下一级目录的职责和进入条件。每个子目录内部有哪些文件、当前版本是什么，由该子目录自己的 `README.md` 维护。
