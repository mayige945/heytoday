# 手工跑通

这里收纳“已经进入手工验证，但还没有资格写成正式系统”的流程材料。

它是 `探索/` 和正式项目之间的中间层：`探索/` 可以发散，`手工跑通/` 必须能按步骤执行，正式项目只吸收被多次验证过的结论。

## 目录规则

- 每个编号子目录就是一个手工任务。
- 每个子目录必须有 `README.md`，写清：目标、输入、步骤、产物、通过标准、沉淀位置。
- 这里放 runbook、短测方法、评估表、阶段结论；正式产品决策仍回到 `PRD_v0.6.md`、`Writing_Kit.md`、`prompts/` 或 `tts/`。
- 稿件正文继续放 `稿子/`；生成音频和请求记录继续放 `音频/`；这里不搬大产物，只做索引和流程控制。
- 一个手工任务连续跑通 3 次，并且没有靠临场判断救场，才考虑沉淀到正式 prompt、脚本或 PRD。

## 当前主线

1. `01_选题与初稿/`：从候选新闻到无音频导演的正文初稿。
2. `02_音频导演稿/`：把正文改成可 TTS 的口语化、带停顿、带轻情绪导演稿。
3. `03_音频生成与评估/`：按当前 MiniMax `line_concat` 路线生成音频，并在同一任务里完成短测、自动预检、人工复听和小修。
4. `04_沉淀正式项目/`：把稳定手工经验写回正式 prompt、脚本和项目文档。
5. `99_历史探测/`：保留已探过但暂不作为主线的后端、标签、音色实验。
