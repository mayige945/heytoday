# 04 发布准备

## 目标

把已经生成并初步核对的单期内容，整理成一组可播放、可分享、可给家长 review 的素材包。这个阶段不改新闻事实和正文结构；只检查对外呈现是否完整、准确、不过度承诺。

## 输入

- `稿子/日期_话题_小学档.md`
- `稿子/日期_话题_minimax导演稿_v*.md`
- `音频/日期_话题_minimax导演稿_v*.minimax.mp3`
- `音频/日期_话题_minimax导演稿_v*.minimax.minimax-request.txt`
- `音频/日期_话题_minimax导演稿_v*.minimax.minimax-request.json`
- 播放封面图片
- 本期成人复听或抽听结论

## 步骤

1. 确认音频文件存在，文件大小和时长合理；若未完整人工复听，必须标明“未完整复听”。
2. 从稿子元数据取 `内容描述（给平台/播放器）`，确认它能提示打开欲望，但不标题党、不提前收口、不外显尚未稳定的“爸爸和孩子/父子/儿子”角色关系。
3. 确认播放封面尺寸符合使用场景：横版播放器封面用 16:9；方形专辑封面用 1:1。封面不使用官方赛事 logo、国旗、真实球员肖像或未经授权品牌元素。
4. 准备对外标题、平台描述、音频路径、封面路径、来源说明和风险提示。
5. 检查本期是否有热更新风险；如新闻赛果、时间、人物状态已经变化，先回到 `01_选题与初稿/` 更新稿子，再决定是否重出音频。
6. 把家长/成人复听结论和孩子反馈入口写清楚；孩子真实反馈进入 `孩子反应日志.md`。

## 产物

- 单期可播放素材清单，至少包括：
  - 标题
  - 平台/播放器内容描述
  - 音频文件路径
  - 播放封面路径
  - 新闻出处或来源说明
  - 成人复听状态
  - 已知风险或待 review 点

## 通过标准

- 家长能一眼知道本期讲什么、为什么值得打开。
- 音频文件、封面、描述、来源说明齐全，路径可追溯。
- 平台描述和封面没有过度承诺当前尚未稳定的声音角色。
- 未完整复听、未跑 ASR、热更新风险等限制被明确标出。
- 没有把内部 review 信息、北极星词、制作指令或 TTS 控制词暴露给普通听者。

## 沉淀位置

- 稳定的发布素材格式写回 `prompts/` 或本目录。
- 封面生成和尺寸规则若反复使用，可新增到 `手工跑通/04_发布准备/` 的专门 playbook。
- 涉及产品定位或正式对外口径的变化，回到 `PRD_v0.7.md` 或 `Writing_Kit.md` 判断是否需要更新。
