# 播客边界与证据锚点 v0.2

> 2026-07-07 第二轮复听反馈修订。v0.1 把入口和锚点写得过硬：常规开场不该被一刀切禁用；证据锚点也不能每次都强提示，否则听起来刻意。

## 一句话

音频播客要尊重听众已经习惯的开始和结束信号；方法论只负责让这些信号有内容、不拖沓、不说教。

## 一、入口边界：常规开始语可用

常规开场是被大量听众习惯验证过的心理预期，不应默认禁用。问题不是“欢迎/今天这期/我们先听”这些词，而是它们后面有没有快速进入本期内容。

可用：

- `今天这期，先听一条有点奇怪的汽车新闻。`
- `欢迎收听。今天先从一辆车前面的小箱子说起。`
- `今天先听一个问题：功能很酷，就一定有用吗？`

入口规则：

1. **可以常规**：允许“今天这期 / 欢迎收听 / 今天先听”这类开始语。
2. **必须有内容**：开始语后 1-2 句内给出新闻物件、主题问题或反差。
3. **不要过长**：不要广告腔、长口号、品牌宣言。
4. **不要空转**：不能只有节目名、寒暄、问候，却迟迟不进入新闻。
5. **TTS 首句稳定**：首句不要只有 2-4 字；常规开始语可以和钩子合成一句。

## 二、证据锚点：按陌生度和回收重要性分强弱

不是所有后面会出现的词都要强锚点。锚点强度按“听众是否陌生 / 是否容易忽视 / 后面是否关键回收”决定。

### 轻锚点

适合：常见词、画面已经很清楚、后面只是顺手再提。

写法：自然提一次，不提醒“记住”。

- `车前面那个箱子，叫前备厢。`

### 中锚点

适合：略专业、容易忽视、后面会回收，但不是全篇唯一钥匙。

写法：画面 + 俗名 + 正式名；不说“先记住”。

- `箱底有个能放水的小圆点，也就是排水阀。`
- 结尾回收：`回到刚才那个能放水的小圆点。`

### 强锚点

适合：陌生专业词、关键证据、后面不记住就听不懂。

写法：画面 + 俗名 + 正式名 + 轻微回收承诺。

- `这个词先放在耳朵里，后面我们还会用到它。`

强锚点慎用。每期最多 1 个，且必须真的影响理解。

## 三、出口边界：可以直接结束

结尾可以比 v0.1 更直白。听众习惯明确结束语，它能释放注意力。

可用：

- `今天这期新闻播客就到这里。`
- `好，今天这期先到这里。`
- `今天先停在这个问题上。`

出口规则：

1. 先留下开放问题，不解释答案。
2. 结束语可以直接说，不必每次都做隐喻式收束。
3. 不用爸爸补一段道理。
4. 不用夸孩子问题问得好。

## 四、导演稿自检

1. 开头是否符合听众常规预期，并在 1-2 句内进入新闻？
2. 本期有没有必须回收的证据？如果有，它该是轻锚点、中锚点还是强锚点？
3. 锚点有没有过度提醒，听起来像“请记住考点”？
4. 结尾有没有明确释放注意力？
5. 开始语和结束语有没有变成固定口号？
