# 奥特曼咨询番外

**状态**：MVP 手工跑通

**最近更新**：2026-07-10（补充“可参与 / 可消费信息”选题门槛，排除与听众无直接关系的品牌零售空间）

**适用范围**：`手工跑通-嗨今天/` 主线每日新闻之外、由家长主动触发的定制资讯番外。目录名按约定使用“咨询”，内容定位仍是奥特曼**资讯**：它不改主线日播的选题池、写稿 prompt 或默认音频形态；只有“奥特曼”当天出现值得讲的可信动态时才启动。

## 目标

跟踪奥特曼相关的制作方动态，以及有事实底座的热门讨论 / 文章；有内容才向家长给选题，没有合格材料就明确记录“无成刊”，不为了日更凑一集。候选还必须和听众有直接关系：孩子能参与其中，或能消费到与事件直接相关的公开信息（如可观看、收听、阅读、游玩的内容或可参与的公开活动），而不是只知道一个品牌在哪里开了店。

这类番外首先是**资讯**：先把“谁在什么时候做了什么、会带来什么变化”讲清楚。它同时必须有强互动性：听众要能不断参与判断、选择或补全信息，而不是被动听一段奥特曼百科或制作公告朗读。

## 输入

- `PRD_v0.8.md`、`Content_Safety_Policy_v0.1.md`：仍用于产品宪法、家长决定权和内容安全；本番外不借“定制”名义绕过它们。
- `prompts/小学高年级_写稿提示词_v0.14.md`：首三期只借用其人物关系、反说教和音频边界规则；不直接把主线的选题形态当成番外的固定模板。
- 当天 / 自上次扫描以来的奥特曼来源材料，以及上一次扫描记录（如有）。
- 家长对候选的选择结果；没有选择就不写稿、不出音频。

## 运行边界

- **手工触发，不建自动日更**：家长发出“扫描奥特曼资讯”后才跑一次；扫描时间和范围写入当日记录。
- **家长仍然拍板**：AI 最多给 0–3 个合格选题并说明互动入口，不替孩子决定听什么。只有 1 个合格选题时就给 1 个，不人为补到 3 个。
- **不混入主线**：它是定制番外，不改变“喂今天”主线的一期单主题新闻和主线候选规则。
- **先看听众能否接上**：制作方直营、合作或大型线下空间本身不构成选题理由。只有孩子能不以购物为前提地实际参与，或能从中消费到与奥特曼内容直接相关的公开信息，才继续判断是否成刊。
- **儿童安全仍有效**：沿用 `Content_Safety_Policy_v0.1.md`。员工私生活、未经证实的制作纠纷、攻击性饭圈争吵、剧透型泄漏和恐吓式营销不做选题；若制作方公开动态本身有敏感面，先按家长档 × 年龄档判定。
- **首三期是验证，不沉淀为全局规则**：先把资讯清楚度和互动反应记到运行记录；连续 3 次有稳定证据后，才决定是否新建专用写稿 prompt 或回写通用规则。

## 输入与来源层级

### A 级：制作方 / 项目原始来源（优先）

1. [Tsuburaya Productions 官方 News Archives](https://tsuburaya-prod.com/newslist/search)：新剧、电影、节目、播出 / 上线、主创公开信息，以及听众可实际参与或获取内容的官方活动 / 合作。
2. 圆谷或项目的官方公告、正式播出平台公告、联合出品 / 发行方公告。若来源只指向转载，继续追到原始公告。
3. [圆谷 Fields Holdings 新闻](https://www.tsuburaya-fields.co.jp/news/)：只在其公告直接涉及圆谷制作的项目、内容业务或可公开核验的制作动态时使用，不能把一般企业信息硬讲成作品新闻。

### B 级：本地官方传播与可核验文章（补充）

1. [奥特曼中国官方微博](https://weibo.com/ultramanfilm)：用于发现中国地区的官方播出、活动或授权传播；涉及作品制作事实时，必须回链 A 级或其他原始公告。
2. 有作者、日期、原始引述的媒体报道 / 深度文章：可以成为“大家在讨论什么”的入口，但必须把事实、作者观点和读者猜测分开写。

### C 级：热门讨论（只作线索，不作单独事实源）

社区帖、热搜、评论区和同好文章可以提示“这个话题正在被聊”，但要同时满足以下条件才可能入选：

- 能追到 A 级或 B 级的可核验事实；
- 讨论的分歧能用具体问题说清，不是站队、攻击或猜内幕；
- 孩子能参与“你会怎么选 / 你从哪条线索判断”，而不是只围观谁吵赢。

**默认不成刊**：只有预售、抽奖、单件周边上架、旧物料重发、无新增事实的怀旧帖、未经证实的泄漏或“听说制作组……”这类内容；仅以售卖、打卡或品牌展示为主的直营店 / 快闪空间开业；以及孩子无法参与、也没有可消费信息的线下行业动态。它们可留在扫描记录里，但不进入选题。

## 每日手工步骤

1. **扫描并记源**：按 A → B → C 查当天和自上次扫描以来的新信息；每条线索先记发布时间、原始 URL、来源层级、事实一句话和是否可能只是营销 / 旧闻。
2. **过“值不值得成刊”门**：只有同时满足以下三条才保留：
   - 有新增、可核验的制作侧动态，或有事实底座的热门文章 / 讨论；
   - 孩子能实际参与，或能消费到与该事件直接相关的公开信息；不能只把“店开了 / 商品卖了”当作参与；
   - 能在两句话内讲清“发生了什么”；
   - 能设计至少一个孩子够得着的互动入口。
3. **给家长选题，或诚实无成刊**：
   - 有合格材料：给 1–3 个候选，每个包含“发生了什么 / 原始来源 / 为什么值得听 / 互动怎么玩 / 风险”。只标推荐，不代替家长决定。
   - 没有合格材料：输出“今日奥特曼资讯番外：无成刊”，列出已查来源和不成刊原因；不写稿、不出音频。
4. **写资讯互动稿**（家长选定后）：先给事件骨架，再让互动穿进去。建议用以下顺序，避免把互动做成答题课：
   - 15 秒内给出一个具体选择、画面或反差；
   - 说明 `谁 / 做了什么 / 何时 / 将改变什么`，事实和推测分开；
   - 至少两次把方向盘交给听众，例如“你会先看哪条公告”“如果你在制作组会保留哪一个设定”；
   - 把官方说法、文章观点、同好猜测逐层标清，不把热门讨论当结论；
   - 结尾留一个轻量开放选择或未解问题，不升华成教育结论。
5. **音频导演与生成**：仍按 `02_音频导演稿/` → `03_音频生成与评估/` 跑。互动优先的验收点是：孩子的台词承担真实猜测 / 选择 / 反驳，爸爸不把答案讲完；首三期先试行“每 60–90 秒至少一次可参与动作”，再按真实反馈决定是否保留这个密度，不要求孩子答对。
6. **复听与反馈**：成人复听除通用六项外，额外记录“哪一次互动让孩子接话”“哪一次解释让孩子断线”“他是想继续猜还是只想要答案”。真实反馈写入 `孩子反应日志.md`，再决定次日要不要继续扫描。

## 产物与命名

- **扫描记录**：`奥特曼咨询番外/运行记录/日期_奥特曼资讯扫描.md`。首次实际运行时再建 `运行记录/`，每次都保留“有候选 / 无成刊”的结论。
- **选题 / 正文**：`稿子/日期_奥特曼资讯番外_话题_小学档.md`。元数据标明“定制资讯番外”、来源层级、事实 / 观点分层和互动设计。
- **导演稿 / 音频 / 评估**：沿用现有 `稿子/`、`音频/` 命名与 `手工跑通-嗨今天/02_音频导演稿/`、`手工跑通-嗨今天/03_音频生成与评估/` 的 request、SenseVoice、成人复听产物；文件名保留 `奥特曼资讯番外` 以便与主线分开。

## 通过标准

- 当天无合格材料时，明确无成刊，没有为了保持频率硬凑选题。
- 每个候选都有原始来源、清楚事实、孩子可参与或可消费的信息，以及可参与的互动入口。
- 正文先说明事实，再讨论观点；不把社区热度、主观文章或传闻念成制作方结论。
- 互动是内容推进器，不是知识抢答：听众不答也能听懂，答了能改变下一轮对话走向。
- 爸爸仍以提问和递球为主；没有说教、标准答案或“你应该”。
- 本番外不因定制身份放松音频生成、SenseVoice 预检和成人复听要求。
