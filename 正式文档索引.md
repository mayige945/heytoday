## 三、会话结束文档检查

每次会话结束前，如果本轮做过以下任一动作，必须跑一遍本节：

- 改了 prompt 或写稿规则。
- 改了选题标准、新闻源范围、候选格式。
- 改了音频导演方法、开场/结尾/证据锚点规则。
- 改了 TTS 脚本、MiniMax payload、`pronunciation_dict`、ASR/评估流程。
- 产生了新的稳定经验，下一期大概率还会用。
- 发现某个 README、AGENTS、CLAUDE、Writing Kit 的当前指针落后。

检查问题：

1. **这是临场样本，还是稳定规则？**  
   只出现一次且不确定，先写进手工记录或孩子反应日志；重复出现、能描述、能执行，才沉淀到正式文档。
2. **应该进哪一层？**  
   产品决策进 `PRD_v0.7.md`；安全边界进 `Content_Safety_Policy_v0.1.md`；写稿规则先进当前 prompt，稳定后同步 `Writing_Kit.md`；手工流程进 `手工跑通/`；脚本行为进 `tts/README.md`。
3. **有没有下游指针要同步？**  
   根索引只查一级入口：本表、`AGENTS.md`、`CLAUDE.md`、`手工跑通/README.md`、`tts/README.md`。进入一级目录后，再由该目录 README 决定是否同步下一层。
4. **旧版会不会误导？**  
   旧版可以保留，但要标“已归档”或确保当前入口不再指向它。
5. **有没有必要新增正式文档？**  
   只有当同一类规则已经跨 2-3 个文件重复、且继续塞进现有 README 会降低可读性时，才新增正式文档。新增在某个一级目录内部时，先登记到该目录 README；只有新增一级入口时，才登记到本索引。

## 四、快速 grep 清单

收尾时优先扫这些旧指针和高风险词：

```powershell
Select-String -Path "AGENTS.md","CLAUDE.md","Writing_Kit.md","正式文档索引.md","手工跑通/README.md","tts/README.md" `
  -Pattern "PRD_v0\\.6|当前.*v0\\.10|当前运行版：`v0\\.10|v0\\.3|早餐桌慢聊感|场景感：早餐桌|小的 `pronunciation_dict` 测试|正式名待定"
```

如果命中的是历史记录、版本记录，可以保留；如果命中的是“当前”“输入”“主线”“通过标准”，必须修。需要继续检查某个一级目录内部时，先进入该目录索引，再按该目录自己的 README 往下一层查。
