## 概览

「喂今天」是一档给小学高年级到初中孩子的每日新闻播客。

表面上，它替家长接走"每天找素材、想怎么跟孩子聊"的备课负担，填上"没有适合这个年龄的新闻"这个市场空白——这是让人**今天就装上**的理由。

但它真正要做的，是一件更长远的事：通过每天一件真实世界的事，**在不说教的前提下，让孩子的思维方式和性格自己长出来**。我们不替父母陪孩子，只给一个家庭每天一个值得一起听的由头。我们只交付内容；成长不是我们当场交付的功能，而是这一切的目的和标尺——我们交付内容，成长交给时间。

## 一、问题陈述（为什么做）

**表层（让人今天就用的理由）**

在重视教育的家庭里，早餐桌上常常已经在发生一件好事：家长跟孩子聊新闻。但它被两件事卡住：

- **家长侧（备课负担）**：要聊得好，家长得提前找素材、想怎么讲。每天这样准备是负担，难以持续。
- **供给侧（内容缺失）**：没有适合这个年龄的新闻内容。新闻都是给成年人做的；孩子常听的有声书，家长又嫌没营养。这块市场是空的。

**深层（为什么值得长期做）**

小学高年级到初中，是孩子开始形成思维方式、也开始认真理解世界怎么运转的年纪——他们对信息的胃口在涨，自己也想知道世界上正在发生什么。可现有的东西都不接这个需求：课本教的是答案和知识点，不是怎么思考；成人新闻是给成人的；没营养的听书只是打发时间。一个能在课本之外、用真实世界喂养孩子思维与好奇的东西，是缺的。

表层是**入口**（让家长今天就装上它），深层是它**为什么值得一直做下去**。下一节"核心机制"把两层连起来：我们怎么用一档每天的播客，真的够到深层那件事。

## 二、产品定位（我们做什么）

**一句话**：一档每天值得被全家讨论的儿童新闻播客。我们只造内容本身。

围绕这条主线，三件事各归其位：

- **引擎 = 实现层，不是产品定位**。把内容能力做成可复用，是内部的工程选择。
- **引发讨论的执行 = 不在范围**。引发讨论是好内容的自然结果，不是我们要建的功能，v1 不做、不测。
- **成长 = 北极星**。它是我们做这件事的理由，也是判断"什么是好内容"的最高标尺；但它指方向，v1 不执行、不考核。

## 三、核心机制：如何在不说教的前提下，影响孩子的思维与性格

这是整个产品的脊椎，也是之前几版最缺的一块。

**我们不教思维，也不教性格。我们创造它们自己长出来的条件。** 成长来自五条机制，全都嵌在每天的内容里，没有一条是"教"：

1. **看见一个值得想的世界（曝光）。** 每天一件真实、复杂、没有标准答案的事。孩子反复接触"世界是有意思、值得琢磨的"——这本身就在养好奇心，也在打一个底：信息不是用来背的，是用来想的。
2. **示范，而不是讲授（模仿）。** 爸爸不给结论，而是当着孩子的面"演"一种思维的样子：问为什么、说"我也不知道，我们一起想想"、找证据、想想别人那边的道理。孩子学会一种思维方式，靠的是模仿一个他在乎的人怎么想，远多于被告知"你要批判性思考"。第一性原理、系统思维、概率思维，不当名词教，而是活在爸爸"怎么面对一件不懂的事"的样子里。
3. **问题大于答案 → 让孩子自己想（主动推理）。** 节目抛开放问题、不收口，于是真正在动脑的是孩子。思维是练出来的，不是听出来的——每天被邀请推理一次，日积月累，那块肌肉就长出来了。
4. **真实的人，而不是美德说教（榜样）。** 性格不当美德教（"你要勇敢/坚持"）。它通过新闻里真实的人被看见——一个真的在面对困难、真的展现了韧性或勇气的人。孩子看见"韧性在一个真人身上、在真实处境里长什么样"，这才是性格真正传递的方式：通过被敬佩的真人和故事，而不是规训。
5. **每天 + 一起听 = 时间的累积（长期主义）。** 一集什么都改变不了。但一年的"每天一起想想真实世界"，会重塑孩子面对信息、问题和困难的方式。成长是积分出来的，不是某天学到的——这就是为什么我们要的是一年后那个孩子，不是今天学了什么。

**这五条里没有一步是"教"。** 也正因如此，后面几条宪法和"成长只当北极星、不考核"这两个决定，都不是凭空定的，而是从这套机制**直接推出来**的——一旦开始教、开始给答案、爸爸一旦变成老师，这五条机制立刻全部失效。

逻辑链一句话串起来：真实的缺口（问题，两层）→ 一档值得讨论的每日播客（产品）→ **它凭什么有用（这套机制）**→ 所以才有这几条宪法 → 所以成长只当北极星、不测。

## 四、长期目标：思维与性格（北极星）

机制要养的，是两类东西。它们不是课程表、不是要逐项打钩的目标，而是上面那套机制长期作用下，孩子身上会慢慢长出来的样子。

**思维方式（怎么想）**：第一性原理、系统思维、批判性思维、科学思维、概率思维、长期主义、创造力、同理心、换位思考、证据意识。它们不当名词教，而是活在爸爸"怎么面对一件不懂的事"的示范里，和孩子被邀请自己推理的过程里。

**性格养成（成为什么样的人）**：好奇心、心理韧性、勇气、自律、坚持、延迟满足、乐观、合作、尊重、责任感。它们不当美德教，而是通过新闻里真实的人——真的在面对困难、真的展现了这些品质的人——被孩子看见。

再说一次：这些是**北极星**，是判断"什么是好内容"的标尺，**不是 v1 要执行的功能，也不是 v1 要考核的指标**。

## 五、产品宪法（从机制推出来）

下面这几条不是孤立的规矩，而是第三节那套机制的**直接推论**——每一条都在守护"不说教也能成长"这件事。一旦违反，机制就失效。

1. **真实世界永远是节目的入口。**（机制 1：没有真实世界，就没有"值得想的世界"可看。）
2. **首先要有趣。** 一档孩子忍着听完的节目，再"有营养"也是失败的。（机制 5：不有趣，孩子不会每天回来，长期累积就无从谈起。）
3. **教育必须隐身。** 禁止"今天告诉我们……""所以我们要……""小朋友应该……"。成长必须自然发生，绝不直接讲授。（整套机制的底线：一开口教，五条全废。）
4. **问题比答案更重要。** 节目负责提出值得思考的问题，不负责提供标准答案。（机制 3：给了答案，孩子就不用自己想了。）
5. **爸爸不是老师，只是和孩子一起探索世界的人。**（机制 2：爸爸一旦变老师，"示范怎么想"就塌成"告诉你答案"。）
6. **我们只对内容负责。** 引发讨论、促成陪伴，是好内容的自然结果，不是我们要建的功能。
7. **我们瞄准的是一年后的那个孩子，但只考核今天的内容。** 成长是方向（北极星），不是 v1 的执行项，也不是 v1 的指标。（机制 5 的必然：长期积分的结果，没法在一集里执行、也没法从孩子生活的其它影响里摘出来单测。）

## 六、目标用户（为谁做）

同一个使用场景里的两个角色，不是两类要二选一的用户。

- **孩子（消费者，也是成败信号）**：小学高年级到初中，约 9–15 岁。对世界有好奇、信息胃口在上涨、已经有听音频的习惯。这个跨度按 2 档处理（见"年龄分档"）。
- **家长（决策者 + 把关人，也是真正"疼"的人）**：重视孩子在课本之外的认知、本来就愿意跟孩子聊、但被备课压力和"没有合适材料"挡住的家长。家长买单，也是内容的最后一道闸。

**明确不是谁**：不是"每一个家庭"（窄而真实的人群，不是大众市场）；不是低龄幼儿（内容密度、话题、呈现都不同）；不是想给孩子找纯娱乐或哄睡内容的家长。

## 七、内容选择与安全

安全不是事后过滤，而是选择架构本身：**到达孩子的每一期，都是人选过的。**

**最高原则**：AI 不决定孩子听什么，家长决定。AI 负责推荐和给理由，拍板的永远是人。

### 候选与推荐

- AI 每天产出 **3 个候选话题**，来自真实世界，彼此不同（不同领域、不同口味，不是同一条新闻的三个变体）。
- 其中 **1 个标"推荐"**。信任系统的家长一眼点推荐那个就走；想自己挑的有另外两个兜着。标推荐不等于 AI 自动选——家长仍要点一下，什么都不点才走常青稿。
- 每条配**一行理由**：领域标签 + 一句具体"勾子"。理由是给家长看的，可以比节目本身更直白地说"这条好在哪"，但**不用教培话术**、**不承诺学习成果**（与"成长不执行不考核"一致）。勾子自带"没有标准答案、有点出乎意料"的味道，但不说"这能培养什么"。

示意：

- 〈推荐〉世界 · 有个国家做了个实验：给所有人每月发一笔钱，想看会发生什么——结果跟大多数人猜的不一样。
- 科学 · 科学家想给珊瑚"搬家"躲开变暖的海水，可有人担心这会惹出新麻烦。
- 社会 · 一座城市把一条主干道禁车一个月试试，结果市民吵翻了。

### 时机

- **前一晚选、连夜生成、早餐打开即播。** 家长晚上花十秒挑好明早的话题，系统连夜生成。
- **早餐零等待是硬约束**——"音频何时就绪"反过来决定了"家长何时选"，所以选在前一晚。
- 配一个**傍晚轻提醒**。这一步本身，就是把家长从前那件有压力的"备课"，原地换成了晚上十秒挑一个。
- 家长忘了选 → 早上走**常青稿**兜底。
- **实时性要求不高**：不追当天最热，从前几天沉淀里挑真正值得聊的事——质量和安全都更可控。新闻源因此**不必做实时抓取**。

### 红线与敏感话题（框架）

内容分三层：

- **红线（永不进池，不可调）**：图像化的暴力与血腥、性内容、自杀方式、仇恨煽动等。连候选都不出现。这是产品的安全底座，不交给家长。
- **敏感但重要（可进，带处理）**：战争、灾难、死亡、犯罪等。完全屏蔽反而假。进池时给家长标"偏沉重"提示，并约束 Narrative：讲影响与人的应对、不讲血腥细节，给抚慰和"人们在怎么帮忙"的出口，不渲染恐惧。
- **默认可聊**：绝大多数科学、社会、文化、经济类话题。

**家长一个总开关，三档**，只调中间那层：

- **保守**：敏感话题基本不进候选。
- **标准（默认）**：敏感话题可进，带提示、按准则处理。
- **开放**：敏感话题正常进、提示弱化。

设置时拨一下即可。**实际判定 = 家长档 × 年龄档**，系统自己算（同样标"标准"，给初中的可比给小学多放一点）。

> 具体的"红线清单 + 敏感话题清单 + 怎么聊的措辞准则"，细到需要单独一份文档，**待后续起草**。本节只定框架与开关。

## 八、节目形态与角色

**一期 = 一条主题新闻。** 家长从 3 个候选里选 1 个，那一条就是当天这期的全部，10–15 分钟围着它展开。3 个候选是给家长挑的范围，不是一期讲 3 条。

**为什么单主题、不是多条速览**：单主题才容得下宪法（有空间慢慢聊、追问、留开放问题），信息密度对这个年龄也刚好。多条速览天然在"给信息/给答案"，容易滑回"播报"，而且要么由 AI 定多条（违反"AI 不自动选"）、要么由家长定（正是要消灭的备课负担）。"想知道世界上在发生什么"这个需求是真的，归宿是 MVP 之后的一个**可选尾巴**：单主题深聊完，末尾一句话带过"今天还发生了 X、Y"，很可能只给初中档。先不做。

**角色：固定爸爸 + 孩子**（有意选择）。不考虑隔代。节目里的两个声音是孩子去听的一对"角色"，不是去模拟某个真实的爸爸。爸爸体现的是一种**思维的样子**——好奇、爱问为什么、肯说"我也不知道，我们一起想想"、看证据再判断；而**不是有答案的权威**（后者会让他塌成"老师"，破坏机制 2 与宪法第 4、5 条）。给 Narrative 的尺子：**爸爸的台词里，问句要多于结论。**

## 九、年龄分档

**2 档**：小学高年级 / 初中。简单、有效、MVP 好验证，正好卡在认知从具体到抽象的转变上。

分档分的是**同一话题的不同"讲法"**，不是不同话题。话题本身和那几条宪法跨档都不变。

| 维度 | 小学高年级（约 9–12 岁） | 初中（约 13–15 岁） |
|---|---|---|
| 词汇 / 句子 | 常用词、短句、具体 | 可用抽象词、复合句 |
| 抽象层级 | 从具体故事、人物、例子切入 | 能上到系统、机制、原理、权衡 |
| 时长 | 偏短（起点 ~8–10 分钟） | 可长些（起点 ~12–15 分钟） |
| 提问类型 | "为什么会这样？""换你会怎么做？" | "换个立场呢？""代价是什么？""有没有别的解释？" |
| 语气 | 亲切、有趣、带点惊奇 | 平等、被当回事，绝不哄小孩 |
| 同一话题的切入 | 通过一个人物或故事 | 通过利害关系或背后的系统 |

**如何切换**：按**年级/学段**设定（家长设置时填学段，比年龄更准、边界不含糊：六年级归前档、初一归后档），随升学自动进档；再给家长一个"太简单/太难"就上下微调一档的能力，兜个体差异，不必测评。不按集切换。

**是否从 2 拆到 3**：暂按 2 档。MVP 之后看观察到的年龄差异有多大再定。

## 十、实现层能力（Engine）

以下五项是"怎么造"，属于实现层，不是产品定位。

1. **Content Intelligence**：当前唯一工程化的模块。通过人工触发的本地 CLI 从 14 个首批采集单元增量采集，完成清洗、去重、事件聚类、价值判断、双年龄档适配、安全初筛与分级（红线/敏感/默认），输出合格新闻池。它不自动推荐、不替家长选题，也不包含后台定时调度。
2. **Cognitive（推荐）**：每天生成 3 个带理由的候选（标 1 个推荐），结合存档决定强化节奏。由北极星指向的方向引导内容选择，但不直接讲授、不量化成固定比例、不考核成长。
3. **Narrative（编剧）**：把选定话题写成**单主题**稿，遵守两把尺子（爸爸问句多于结论；按年龄档套用第九节讲法），敏感话题按第七节准则处理。它落地的正是第三节那套机制——示范、开放问题、真实的人。
4. **Family Personalization**：按学段（定年龄档）、兴趣、偏好、声音等，让每个家庭有自己的版本。年龄分档在这里落地。
5. **Voice Performance**：角色切换、停顿、语气，最后调 TTS 生成音频。可替换，不影响上层。

## 十一、技术架构

```
新闻源（人工触发 CLI 采集，不后台定时）
  ↓
Content Intelligence（独立模块：采集 + 去重聚类 + 安全初筛/分级 + 双年龄档适配 → 合格新闻池）
  ↓
Cognitive（当前手工：结合存档 → 3 个带理由的候选，标 1 个推荐 + 决定强化节奏）
  ↓
家长前一晚选定（人工闸；AI 不自动选、不自动发）
  ├─ 选了 → Narrative 写单主题稿 → 安全过滤 → 默认发出
  │         （家长可预览/否决；或"每期我先审"先审后发）
  │         → Voice Performance 连夜生成音频 → 早餐打开即播
  └─ 没选 → 常青稿顶上（早餐打开即播）
  ↓
每期文稿 + 元数据（日期、话题、触及的思维/性格点）存档 → 回流到 Cognitive 的推荐
```

输出形式（RSS / App / 未来载体）可替换，上层不变。原"全自动分发"已调整为"AI 推荐 + 家长前一晚把关"——分发前必有人工这一步，或由常青稿兜底。

## 十二、成功标准

产品化后，v1 只看一件事：**孩子愿不愿意每天回来听。** 指标是完播率、连续收听天数、次日留存。

既然"备课压力"是家长买单的核心理由，另外轻轻量一下它有没有被接走。这不是硬指标。

我们**刻意不测**：有没有引发讨论（不在范围），以及孩子有没有成长（北极星，无法归因到一档节目）。

> 注：MVP 阶段是 N=1（见下节），"指标"主要靠**现场直接观察**而非统计——这套收听指标在产品扩出去之后才真正用得上。

## 十三、MVP 与验证计划

**MVP = 在自己家、对单个孩子、AI 全流程生成。**

把场景缩成"自己家、一个孩子"，改变了生成方式的逻辑。原本顾虑"孩子不爱听时分不清是点子烂还是稿子烂"——但那是针对看不见的外部用户、只能从冷数据反推。现在听众是自己的孩子，反应当场看得见，质量这个变量不再需要靠人力摁住来反推。所以这里用 AI 全流程是成立的。

**成立的前提（要写明）**：全 AI 在这里成立，不是因为"终点是 AI"，而是因为**本人在现场、能当场判稿和反馈**——人没退出，而是从"写手"变成"裁判 + 教练"。这个前提随单孩子设置成立；**扩到看不见的外部家庭时必须重审**，那时"半人肉"（AI 起草、人重度改写）的顾虑会回来。

**MVP 的真实目标顺势升级**：不只是验证"孩子爱不爱听"，而是借这个高频循环（AI 出稿 → 孩子真实反应 → 改 prompt/设定 → 次日再看），调出一套能稳定产出"好稿"的 **prompt 与规则**，也就是 Narrative 的雏形（即第三节那套机制如何落进 prompt）。

**一个已知偏差**：N=1 会把 prompt 带偏——容易调出"特别懂我儿子"的版本。MVP 阶段没问题，但要心里有数：优化出来的东西里，一部分是"好内容"的通用规律，一部分是"我儿子的口味"。扩到更多孩子时要把两者分开——通用的留进 Narrative，私人的归到家庭个性化。别把"我儿子爱听"直接等同于"产品成立"。

**当前工程验证范围**：只验证新闻采集模块能否以手动 CLI 独立运行，并稳定输出合格新闻池。TTS、Podcast 平台、自动推荐和自动分发仍不做系统化验证。

## 十四、范围与不做什么（v1 边界）

**v1 做**：一档每天的单主题儿童新闻播客；AI 推荐（3 选 1）+ 家长前一晚把关；2 档年龄讲法；文稿存档与基础推荐参考；常青稿兜底。

**v1 不做**：多形态扩展（音箱/玩具/车载/机器人/视频）；新闻以外的输入源；速览/覆盖面（转 MVP 之后、可能仅初中档）；"引发讨论"的功能与测量；"成长"的执行与考核；引擎对外作卖点；后台实时抓取与定时调度；自动推荐、自动选题、自动写稿、自动音频与自动发布；隔代角色。

## 十五、开放问题（待决 / 有触发条件）

- **副标题**：正式名称已定为「喂今天」，副标题待定。
- **红线 / 敏感话题的详细内容策略文档**：框架已定，详细清单与措辞准则待起草（单独文档）。
- **常青稿库容与轮换**：MVP 先备 3–5 个主题 × 两档；真正库容等 MVP 看到兜底实际多久触发再定。
- **速览尾巴**：MVP 之后再定，可能仅给初中档。
- **年龄是否从 2 档拆到 3 档**：数据驱动，MVP 之后再定。
- **扩到外部家庭时**：重审"AI 全流程"的前提，届时"半人肉"顾虑回来。
